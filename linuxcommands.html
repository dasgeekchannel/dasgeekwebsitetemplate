Section 1: Linux Manual

The Linux manual is known as ‘man’ pages. It can be accessed via the terminal and contains a wealth of information for new users and experts. There are so many commands in Linux its impossible to memorize them all. Thankfully the man pages exist so you don’t have to remember them all. The one thing I wish I used more when I first started Linux was the man pages vs. relying solely on the Internet.

To use the man pages simply:

1. Open Terminal

2. Type man [search term]
Ex: man users

 

Navigation

Navigating the man pages just requires knowing a few simple shortcuts.

/

 

Type / while in a man page to search for a keyword.

Ctrl + F

 

You can scroll with your mouse or Ctrl + F to move forward

Ctrl + B

 

You can scroll with your mouse or Ctrl +B to move backwards

Q

 

Hit q to quit out of terminal and return back to the terminal.

 

 

Man Page Commands

man 5 passwd

 

Notice the number 5. This represents section 5 of the passwd man pages. Use the number variable to look at specific sections

man -k

 

The -k is a keyword search. So man -k users would search for the keyword ‘users’ in all the man pages.  

man – af

 

The -af searches in just the title of the man pages for a specific word. For example man -af users would search for any man pages that contain ‘users’ in the title.

man -h

 

Type man -h for a full list of all the possible ways to search and display pages from the man pages.

 

INFO hyper-link capable man page alternative

 

Another option that contains hyper-links and easier navigation is ‘info’. Info can be used in place of man and contains many of the same articles. Simply substitute man for info to search.

 

To use the info simply:

 

1. Open Terminal

 

2. Type info [search term]

Ex: Info users

 

 
 
 
 

Section 2: Basic Terminal Commands

Having some basic terminal commands down will help you start to feel the true power of the terminal. Here are some basic commands that you’ll find yourself using over and over again. After any command type man and the command to get more info. Ex: man ls

 

ls

 

ls is a great command for listing a directories contents. Simply type ls at the terminal prompt and you will see a listing of directories.

ls -l

 

ls – l will help you list out those directories in a long list. Other ls commands include:
ls -la = long list with colorized folders based on type
ls-d (Directory) shows you what's inside that file like Documents
ls - F puts a special character by the files to designate if they are a executable, directory, etc.

cd

 

cd stands for change directory. This allows you to navigate to directories. Use ls to get a list of directories and to enter one like Downloads type cd Downloads and hit ENTER. Now you can type ls again to list out the contents of Downloads.

history

 

Type history in the terminal to see a list of all the terminal commands you’ve entered recently. This is extremely useful for recalling commands you need to use again.

pwd

 

Print working directly. To know what directory you’re currently in just type ‘pwd’ and you will see the path that you’re currently working in.

Touch

 

Creates an empty file in a directory. Ex: Touch File1 will create an empty file called ‘File1’

grep

 

Provides an advanced search function to find specific files, words, etc.

Ex: grep dasgeek /etc/passwd would search for dasgeek in /etc/passwd or you could use a /R for recursive to look through all the files in that directory.

find

Another search function for files.

du -h

 

Easy to remember du stands for data usage and the -h makes it human readable. Find the amount of space a directory is using.

clear

 

Clears the terminal screen so your back to a nice blank terminal just like when you first open it.

apt-get update

 

Used in Debian based distros to update your local cache of the packages.

 

yum update for Red Hat based distro’s

apt-get upgrade

 

Used in Debian based distros to upgrade software on your Linux system including the system itself.

 

apt-get dist upgrade will upgrade your distribution to the latest version. ← use with caution. Ex: Upgrade from Ubuntu 16 to Ubuntu 17.10

 

yum upgrade for Red Hat based distros

 

 
 

Debian/Ubuntu specific commands

 

apt-cache search

 

Will let you search for specific software packages in the APT repository. Ex: apt-cache search dropbox

apt-cache install

 

Will install specified software package using sudo for admin rights for install. Find a program with apt-cache search and install it with apt-cache install.  

yum search

 

Will let you search for software packages in Red Hat based distros

yum install

 

Will install specified software package in Red Hat based distos.

 

 

 

 

 

 

 

 

 

 

 

 

Terminal Tips

 

[CTRL] + a

 

Moves to the start of the command in the terminal

[CTRL] + e

 

Moves to the end of the command in the terminal

[CTRL] + r

 

Reverse search for commands

History

 

Type ‘history’ in the terminal to show history of commands used.

[Up] Arrow / [Down] Arrow

 

Pressing the up and down arrow on your keyboard will cycle through prior used commands in the terminal.

[Tab]

 

If you start typing out a command in the terminal or directory you can double tap the [Tab] key and Linux will fill out the rest of the command for you. Ex: Type: cd downl  (now hit tab twice and it will fill it out to the directory ‘downloads’.

[CTRL] + c

 

End process running in the terminal

[CTRL] + [Alt] + t

 

Open a terminal in Ubuntu based distros

 

 

 

 

 

Installing Files Outside The GUI

 

 

Debian/Ubuntu packages use .deb

Redhat uses RPM packages .rpm

Arch uses TAR

The above are the file types used much like Windows uses a .exe. There are also user created packages. For instance Debian/Ubuntu uses PPA (Personal Package Archive) where users submit packages for installation. Additionally when you install a distro like Ubuntu some official PPA’s from Canonical will be automatically installed. Arch based distros utilize AUR (Arch User Repository) for community compiled software.

For new users I recommend sticking with the GUI (Graphical User Interface) for awhile until you’re comfortable with Linux. Just open the store and click install. Its just that easy. However, you will soon find yourself eventually wanting a piece of software that you can’t get in the software store. Let’s take a look at our options:

 

Install using a flatpak or snap. If your distro supports flatpak and snap packages you can universally install these and they include the dependencies. This is becoming more popular so you may see this option on the software website.

 

Another option you may see is to install a PPA and then install the software. This essentially adds an additional software source and allows you to download the software once the PPA is installed. This can be dangerous if you add untrusted PPA sources. If you’re on Arch you can enable AUR packages and they come with the same potential risks as an unknown PPA.

 

Finally, you can download a tar.gz, .deb, or .rpm (there are others) for your respective distro and install those. We will cover a few of those formats  in more detail below:

 

.deb

A .deb file designates its a Debian based file and can be downloaded and installed just like a typical executable file in a Windows environment if you’re on a Debian based distribution (Ex: Debian, Ubuntu, Mint, etc.). Simply double click the .deb file and the software manager will pick it up and install it along with any dependencies. You can also alternatively use GDebi to install the package verse using the software store in case you get an error.

 

Finally you have the option of installing .deb files via the terminal using dpkg command.

Ex: dpgk -i ‘/path of file/file’

 

*This method along with RPM method will not handle dependencies. Meaning you may need to manually install additional software to get dpkg or RPM to work.

 

.RPM is very similar in that you can install the files but use the RPM commands.

 

 

tar.gz

This is a compressed file format. It is common in the Linux world. You can generally uncompress the file via the GUI interface once its install by extracting the compressed package. After extraction it is recommended that you read the ‘readme’ or ‘installation instruction’ file inside. If they do not provide one then you can generally use the following commands to install it from the terminal.

 

Step 1: Navigate to the directory where the extracted file is located. Ex: cd /Downloads/File1

Step 2: ./configure

Step 3: make install

 

During the ./configure phase pay attention to the output to see if you’re missing any dependencies. You will need to install those dependencies in order to complete the install.

As mentioned, it is better to stay away from these type of installs as they can change configuration files as well. Better to find an alternative software to install (alternativeto website has many options). Or use a SNAP or FLATPAK which are universal to any distro that supports them or a package that is made for your distribution (ex .deb for Ubuntu). These are also much easier to uninstall.

Additional Tips & Tricks:

‘/etc/apt/sources.list is where your repositories are stored. You can remove or add repositories here or via the GUI.

 

If you maintain multiple computers you can create a local repo.

*Ubuntu Instruction:

- apt-get install apt-mirror

- creates a local repo in ‘/etc/apt/mirror.list’

- sudo apt-mirror will download all of the repo data locally to your machine (50+GB)

- Create a ftp connection so other computers can download from the location ‘/var/spool/apt-mirror’

- Will need to updateBookmark 1 local sources on each machine to point to this new local repo. However, updates across all the machines will be much faster and not require a direct internet connection.

Scripting

The power of cat command and > , >>

The cat command is a very common command used in the terminal. It has a variety of uses such as viewing contents of a file or even creating or appending information to a file. It is a great command to learn.

First it will be easier to run these commands and scripting as root. To enable a super user (su) in Ubuntu simply type ‘sudo passwd root’ then set password for root. Now type su and enter new password to remain in root. This allows you to run updates, installs, etc. without having to type your password in each time. However, this can be very dangerous as you can make permanent fatal changes to your files or system. Use with caution.

Create a file using >

In terminal type: echo “dasgeek rules” > test.txt

Now type: cat test.txt  

This should display the ‘dasgeek rules’

We can also append some text to the test.txt using  echo and >>

In terminal type: echo “dasgeek is also my friend” >>test.txt

Now when you cat test.txt you will see the second line appended to the file

You can also create a file using touch command

In terminal type: touch test2.txt

Now if you ls you will see test2.txt but it will be a blank .txt file.

You can use echo and > to add text into it.
